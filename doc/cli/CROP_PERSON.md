### Задание: Разработка скрипта для вырезания людей и вписывания в квадрат

Необходимо написать Python-скрипт, который обрабатывает изображения с помощью модели сегментации SAM3.

**Основные требования:**

1.  **Входные данные:**
    *   Скрипт принимает путь к файлу или папке.
    *   Если передана папка, скрипт должен рекурсивно найти все изображения внутри.

2.  **Работа с моделью:**
    *   Используется библиотека `ultralytics` (класс `SAM3SemanticPredictor`).
    *   Путь к модели: `models/sam3.pt`.
    *   Промпт для поиска: `"person"`.
    *   **Важно:** Модель должна загружаться в память только **один раз** и только в том случае, если найдено хотя бы одно изображение для обработки.

3.  **Логика обработки (ключевой момент):**
    *   Для каждого найденного человека скрипт получает его маску и ограничивающий прямоугольник (BBox).
    *   **Алгоритм создания результата:**
        1.  Вырезать из *оригинального* изображения прямоугольную область, ограничивающую человека (обычный кроп прямоугольника).
        2.  Создать новое изображение: **белый квадрат**. Размер стороны квадрата равен большей стороне вырезанного прямоугольника.
        3.  Вставить вырезанный прямоугольник с человеком в центр этого белого квадрата.
    *   **Важно:** Не применять маску для удаления фона внутри прямоугольника. Фон вокруг прямоугольника должен быть белым, а сам прямоугольник вставляется как есть.

4.  **Нюансы обработки:**
    *   На одном изображении может быть несколько людей. Скрипт должен сохранить каждого в отдельный файл.
    *   Если человек выходит за границы исходного фото, при вставке в квадрат он должен располагаться со смещением, сохраняя визуальный центр.

5.  **Выходные данные:**
    *   Формат: JPEG.
    *   Именование файлов:
        *   Если найден 1 человек: `имя_файла_person.jpg`.
        *   Если найдено несколько: `имя_файла_person_0.jpg`, `имя_файла_person_1.jpg` и т.д.
    *   Аргументы консоли: `input_path` (обязательный), `--debug` (сохранять изображение с рамками), `--verbose` (подробное логирование).

**Стек технологий:** Python, PyTorch, PIL (Pillow), NumPy, Ultralytics.

# Функциональные требования

## 1. Входные данные (CLI)
Скрипт должен принимать аргументы командной строки:
- `input_path` (str, обязательный): Путь к файлу или директории.
- `--output_dir` (str, опциональный): Папка для результатов (по умолчанию `./output`).
- `--debug` (flag): Если установлен, сохранять исходные изображения с отрисованными bounding boxами.
- `--verbose` (flag): Включить подробное логирование (INFO уровень).
- `--conf` (float, опциональный): Порог уверенности модели (по умолчанию 0.5).

## 2. Логика поиска файлов
- Рекурсивный обход папки, если передана директория.
- Поддерживаемые расширения: `.jpg`, `.jpeg`, `.png`, `.bmp`, `.webp` (регистронезависимо).
- Пропускать битые файлы с логированием ошибки, не прерывая выполнение.

## 3. Работа с моделью
- **Ленивая загрузка:** Модель инициализировать только после подтверждения наличия файлов для обработки.
- **Однократная загрузка:** Инстанс модели должен быть единым для всего пакета обработки.
- **Промпт:** Использовать запрос `[УТОЧНИТЬ: текстовый "person" или класс-индекс 0]` для поиска людей.

## 4. Алгоритм обработки изображения
Для каждого найденного объекта (человека):
1.  **Получение координат:** Извлечь BBox (x1, y1, x2, y2) из предсказания модели.
2.  **Кроп:** Вырезать прямоугольную область из *оригинального* изображения по координатам BBox.
    - *Важно:* Маску сегментации не применять для альфа-канала. Фон внутри кропа остается оригинальным.
3.  **Создание холста:**
    - Создать новое изображение режима RGB, залитое белым цветом `(255, 255, 255)`.
    - Размер стороны квадрата `S = max(width_crop, height_crop)`.
4.  **Вставка (Paste):**
    - Разместить кроп в центр белого квадрата.
    - **Обработка краев:** Если исходный BBox был обрезан границами фото, центрирование производится относительно геометрии вырезанного прямоугольника.
5.  **Сохранение:**
    - Формат: JPEG (качество 95).
    - Именование:
        - 1 объект: `{original_name}_person.jpg`
        - N объектов: `{original_name}_person_{index}.jpg` (index с 0).
    - Если на изображении людей не найдено — файл не создается (или логируется warning).
